<objective>
Create an automated pricing consistency test suite that verifies the same CAD price renders
on category cards, product pages (PDP), and cart for representative products.

Purpose: The authority pricing overlay system has multiple code paths (category → ProductPrice.vue
with roundTo99, PDP → getFormattedPriceDisplay without roundTo99, cart → formatAdvertisedCadPrice
with roundTo99). Any regression in the overlay, the SEO fallback, or the CAD-detection logic
causes prices to diverge across surfaces. This test catches that before users see stale/wrong data.

Output: `scripts/test-pricing-consistency.js` — runnable via `node scripts/test-pricing-consistency.js`
</objective>

  <action>
Create a Node.js script (no external deps, uses built-in https) that tests pricing consistency.

The script MUST test THREE surfaces for each product:
1. **Authority API** — POST to /api/authoritative-product-prices, extract the price/regularPrice/rawPrice for the product and each variation
2. **PDP SSR payload** — GET the product page HTML, parse the `__NUXT_DATA__` script tag, extract all objects with price/regularPrice fields, verify variation nodes have CAD-tagged prices
3. **Rendered HTML** — Check the `text-4xl` price display area in the server HTML for the dollar amount

For each surface, extract the numeric CAD price. Then assert:
- All three surfaces show the SAME numeric price
- Variation prices match product-level price (for uniform-priced products)
- Prices contain "CAD" tag (authority-overlaid prices include `&amp;nbsp;CAD` or ` CAD`)
- NO original USD prices survive (e.g., no `$249.xx` or `$298.xx` without CAD tag)

Test products (cover multiple scenarios):
1. `usd-aeon-80-salvia-aggressive-inline-skates` (databaseId: 169786) — test-only product, triggers SEO fallback, NO production equivalent
2. Pick 2 real production products from data/products-list.json: one with sale price, one without

Script structure:
```
const BASE_URL = process.env.TEST_URL || 'https://test.proskatersplace.ca';

async function testProduct(slug, databaseId, label) {
  const results = { authority: null, pdpPayload: null, pdpHtml: null };
  
  // 1. Authority API
  // POST to ${BASE_URL}/api/authoritative-product-prices
  // Extract rawPrice from response.products[slug]
  
  // 2. PDP page fetch
  // GET ${BASE_URL}/product/${slug}
  // Parse __NUXT_DATA__ JSON array
  // Find objects with price/regularPrice keys
  // Extract the product-level and variation-level prices
  
  // 3. Rendered HTML price
  // Find text-4xl section in HTML, extract dollar amount
  
  // Compare all three
  // Report PASS/FAIL with details
}
```

Output format: clear PASS/FAIL per product with details on mismatch.
Exit code 1 if any test fails (for CI integration).

IMPORTANT: The NUXT_DATA payload is a flat JSON array where objects use integer keys referencing
other array positions. To resolve an object's field names and values:
- Object keys are string indices into the array (the value at that index IS the key name)
- Object values are indices into the array (the value at that index IS the value)
- Parse objects by iterating entries, looking up both key and value by index

Handle the edge case where the product page may be KV-cached with stale data — the test should
note when it suspects a stale cache (e.g., price in __NUXT_DATA__ doesn't match authority API).
