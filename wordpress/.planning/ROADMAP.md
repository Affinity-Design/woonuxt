# Roadmap: ProSkaters Place US WordPress SEO Program

## Overview

This roadmap starts by fixing truth, governance, and obvious template hygiene gaps before moving into higher-leverage commercial SEO work. It deliberately treats automation as a quality-sensitive delivery mechanism, not the objective itself, so later scale work only happens after the site, the scripts, and the measurement loop are all trustworthy.

## Phases

**Phase Numbering:**

- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions if new live-site issues appear

- [ ] **Phase 1: Governance Baseline** - Lock current truth, owners, KPIs, QA gates, and artifact expectations.
- [ ] **Phase 2: Template Hygiene Fixes** - Remove known duplicate FAQ and shipping-threshold inconsistencies from live templates.
- [ ] **Phase 3: Brand Rollout Closure** - Finish the remaining operational work around the live brand-page system.
- [ ] **Phase 4: Commercial Category Upgrade** - Improve roller skates and inline skates pages and launch the roller-skates hub.
- [ ] **Phase 5: Intent Routing and Redirects** - Define reviewed redirect mappings and deploy them safely.
- [ ] **Phase 6: Answer-First Article Retrofit** - Improve article structures for AI Overview extraction and editorial quality.
- [ ] **Phase 7: Trust and Author Layer** - Standardize author and trust presentation across article templates.
- [ ] **Phase 8: Product Rewrite Readiness** - Build the QA and rollout guardrails needed before scaled product-description rewrites.

## Phase Details

### Phase 1: Governance Baseline

**Goal**: Turn the current SEO plan into an operating system with consistent status language, ownership, KPI review, and QA evidence.
**Depends on**: Nothing (first phase)
**Requirements**: GOV-01, GOV-02, GOV-03
**Success Criteria** (what must be TRUE):

1. Every active workstream has an owner model, dependency set, KPI target, and QA gate.
2. Documentation clearly distinguishes planned work from live verified work.
3. The research pack and project docs point future planning at the right sources of truth.
   **Plans**: 2 plans

Plans:

- [ ] 01-01: Normalize source-of-truth docs and operating vocabulary.
- [ ] 01-02: Define artifact tracking, reporting cadence, and acceptance workflow.

### Phase 2: Template Hygiene Fixes

**Goal**: Remove visible contradictions and shared-template defects that undermine trust in the live site.
**Depends on**: Phase 1
**Requirements**: HYG-01, HYG-02
**Success Criteria** (what must be TRUE):

1. The homepage and related shared surfaces no longer repeat the duplicate FAQ item.
2. Shipping-threshold messaging is consistent and comes from the intended source template.
3. The fixes leave behind a reproducible method for future template hygiene checks.
   **Plans**: 2 plans

Plans:

- [ ] 02-01: Locate and fix the duplicate FAQ source and confirm live output.
- [ ] 02-02: Trace shipping-threshold ownership and fix the canonical template source.

### Phase 3: Brand Rollout Closure

**Goal**: Finish the measurement and retry work required to mark brand automation as live and verified.
**Depends on**: Phase 2
**Requirements**: BRD-01, BRD-02, BRD-03
**Success Criteria** (what must be TRUE):

1. Remaining failed brands are retried or documented with explicit blockers.
2. A representative live brand sample passes content placement and FAQ schema checks.
3. Search Console and rich-results follow-up are attached to the rollout.
   **Plans**: 2 plans

Plans:

- [ ] 03-01: Close the failed-brand queue and validate a live sample.
- [ ] 03-02: Add measurement and rich-results follow-up artifacts for brand pages.

### Phase 4: Commercial Category Upgrade

**Goal**: Upgrade the top commercial category assets and publish a dedicated roller-skates hub page.
**Depends on**: Phase 3
**Requirements**: HYG-03, HYG-04, COM-01, COM-02, AUT-01
**Success Criteria** (what must be TRUE):

1. Roller skates and inline skates pages use US-consistent, transactionally useful copy.
2. The roller-skates hub page is live, linked, and aligned to the commercial keyword cluster.
3. The content update workflow has a documented review sample and rollback path.
   **Plans**: 3 plans

Plans:

- [ ] 04-01: Build the content brief and QA rubric for category upgrades.
- [ ] 04-02: Implement roller skates and inline skates category upgrades.
- [ ] 04-03: Publish and integrate the roller-skates hub page.

### Phase 5: Intent Routing and Redirects

**Goal**: Correct major keyword-to-URL mismatches using reviewed mappings and safe redirect deployment.
**Depends on**: Phase 4
**Requirements**: COM-03, COM-04
**Success Criteria** (what must be TRUE):

1. A reviewed query-to-URL map exists for the highest-value intent mismatches.
2. Redirect dependencies are installed or confirmed before implementation begins.
3. Live redirects return the intended destination and are ready for recrawl requests.
   **Plans**: 2 plans

Plans:

- [ ] 05-01: Produce and approve the redirect map.
- [ ] 05-02: Implement and validate redirects on live URLs.

### Phase 6: Answer-First Article Retrofit

**Goal**: Improve top traffic articles so they answer intent directly and read less like scaled AI output.
**Depends on**: Phase 5
**Requirements**: EDT-01, EDT-02, AUT-02
**Success Criteria** (what must be TRUE):

1. Priority articles open with direct answers that match their target queries.
2. Updated pages reduce repetitive AI-style phrasing and preserve strong internal linking.
3. Any quick-answer blocks validate against the live page structure and promised schema behavior.
   **Plans**: 3 plans

Plans:

- [ ] 06-01: Select priority articles and define answer-first patterns.
- [ ] 06-02: Implement quick-answer automation or editorial retrofit on the first batch.
- [ ] 06-03: Run human QA on updated articles and capture outcomes.

### Phase 7: Trust and Author Layer

**Goal**: Upgrade the partial author system into a clearer trust and E-E-A-T presentation standard.
**Depends on**: Phase 6
**Requirements**: EDT-03
**Success Criteria** (what must be TRUE):

1. Article templates show a consistent byline and useful author destination.
2. Trust-oriented template elements do not break mobile layout or page clarity.
3. Planning language now treats authorship as a refinement project, not a missing system.
   **Plans**: 2 plans

Plans:

- [ ] 07-01: Define the target byline, bio, and trust-block pattern.
- [ ] 07-02: Implement and validate the pattern on representative article templates.

### Phase 8: Product Rewrite Readiness

**Goal**: Create the QA, sampling, and rollback framework needed before any scaled product-description rewrite launch.
**Depends on**: Phase 7
**Requirements**: AUT-03
**Success Criteria** (what must be TRUE):

1. A written QA rubric exists for product-description rewrites.
2. A 10-page sample review process and rollback path are documented.
3. The team can decide to scale or stop based on evidence instead of optimism.
   **Plans**: 2 plans

Plans:

- [ ] 08-01: Define rewrite quality rubric, prompts, and rollback method.
- [ ] 08-02: Validate the sample workflow and go/no-go criteria.
